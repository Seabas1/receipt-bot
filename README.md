# 🧾 RECEIPT_BOT: ML-система распознавания и парсинга чеков

Telegram-бот для автоматического извлечения данных из фотографий кассовых чеков. Бот принимает изображение, проводит его через пайплайн компьютерного зрения и машинного обучения, после чего возвращает пользователю структурированный Excel-файл (`.xlsx`) с наименованиями позиций и их стоимостью.

**Особенность проекта:** Полностью автономная работа без использования сторонних платных API (таких как Yandex Vision или Google Cloud Vision). Распознавание базируется на кастомно обученных моделях.

## 🛠 Технологический стек
* **Бекенд и Бот:** Python, `aiogram` (или `python-telegram-bot`)
* **Computer Vision (Детекция):** `opencv-python`, `ultralytics` (YOLO)
* **OCR (Распознавание текста):** `torch`, `torchvision`, `easyocr`, `pytesseract`
* **NLP (Извлечение сущностей):** `transformers`
* **Обработка данных:** `pandas`, `openpyxl`, `numpy`

# План выполнения проекта
Разработка разбита на 4 основных этапа: от подготовки данных до деплоя интерфейса.

Этап 1: Подготовка данных (Data Preparation)
Для успешного обучения моделей компьютерного зрения требуется качественная разметка.
[ ] Детекция (Image Recognition): Адаптация готового англоязычного датасета для обучения модели поиска самого чека на фото и/или выделения текстовых блоков (BBox).
[ ] Распознавание (OCR) - Синтетика: Написание скрипта для генерации синтетических данных. Генерация изображений строк с различными шрифтами (кассовыми), шумами, искажениями перспективы и артефактами печати.
[ ] Распознавание (OCR) - Real Data: Разметка 200 собранных реальных чеков для валидации и дообучения (fine-tuning) OCR-модели на "боевых" данных.

Этап 2: Обучение моделей (Model Training)
Создание ML-пайплайна для извлечения сырого текста из картинки.
[ ] Модель детекции: Обучение YOLO (ultralytics) на англоязычном датасете для локализации чека и обрезки фона.
[ ] Предобработка: Реализация алгоритмов OpenCV для выравнивания перспективы, бинаризации и очистки от шумов.
[ ] Модель OCR: Обучение/дообучение нейросети (на базе torch / кастомного энкодера-декодера или fine-tuning easyocr/transformers) на смеси синтетических данных и 200 реальных чеков.

Этап 3: Парсинг и NLP (Data Extraction)
Перевод "сырого" распознанного текста в структурированный формат.
[ ] Разработка алгоритма сопоставления текстовых строк.
[ ] Использование регулярных выражений или легковесных LLM/NER моделей (через transformers) для классификации токенов: выделение наименования позиции (Блюдо/Напиток) и привязка к ней числового значения (Стоимость).
[ ] Фильтрация мусорной информации (ИНН, ФИО кассира, НДС, промежуточные итоги).

Этап 4: Разработка интерфейса и интеграция (Bot & Export)
Сборка всех компонентов в единый рабочий продукт.

[ ] Разработка логики бота на aiogram: прием фото, отправка сообщений о статусе обработки.
[ ] Интеграция ML-пайплайна в асинхронный процесс обработки сообщений.
[ ] Формирование .xlsx документа с помощью pandas и openpyxl.
[ ] Возврат готового файла пользователю и очистка временных файлов.
